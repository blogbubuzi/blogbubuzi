    Electron开发中的常见错误

    bubuzi 2/21/2023

    # 编译错误

    # 无法加载本地二进制程序

    Uncaught Error: No native build was found for platform=win32 arch=x64 runtime=electron abi=76 uv=1 libc=glibc
    1

    解决办法：

    1. 在electron-builder中配置external

    如在vue.config.js中

    module.exports = defineConfig({
  pluginOptions: {
    electronBuilder: {
      externals: ['serialport']
    }
  }
})
    1
    2
    3
    4
    5
    6
    7
    1. 若上一步没有成功，则尝试重新构建
    # 进入指定路径
$ cd .\node_modules\@serialport\bindings-cpp

# 重新编译模块
# msvs_version是你的vs版本
# abi通过报错可以知道
# target是你的nodejs版本号
$ node-gyp --msvs_version=2022 rebuild --platform=win32 --abi=101 --runtime=electron --target=16.18.1 --arch=x64 --target_arch=x64
    1
    2

    4

    若一直报node-gyp报错，修改全局安装目录下的相关文件

    • 'node_modules\node-gyp\lib\configure.js'
    • 'node_modules\node-gyp\lib\find-visualstudio.js'

    设置vs安装目录

    $env:VCINSTALLDIR="C:\Program Files\Microsoft Visual Studio\2022\Community"
    1

    打开find-visualstudio.js，发现其中有一项操作，导致一直找不到vs命令行

    path.resolve(process.env.VCINSTALLDIR, '..')
    1

    将其改为

    path.resolve(process.env.VCINSTALLDIR, '')
    1

    即可

    # 打包后出现包未找到异常

    Error: Cannot find module 'serialport'
    1

    解决方法：

    检查package.json文件，对应模块是否存在于devDependencies，若是，将其删除，并重新安装至dependencies中

    原因：

    electron-builder在编译中会将devDependencies中的内容排除
